Salesforce MCP — Setup

Get the Salesforce MCP running on your Mac and connected to Claude Desktop / Cowork or Claude Code.

macOS only No coding experience needed ~30 minutes

Before you start — one-time setup

You will need three things before running the technical steps:

  1. A GitHub account (free) with multi-factor authentication enabled
  2. Access to the WillowTree Salesforce MCP repository (or use the alternative)
  3. About 30 minutes of uninterrupted time

A. Install an authenticator app (do this first)

GitHub now requires multi-factor authentication (MFA) for all accounts. You'll need an authenticator app on your phone before you create the GitHub account — otherwise you'll be locked out during signup.

Install one of these on your phone (any of them will work):

  • Authy — App Store · Google Play
  • Google Authenticator — App Store · Google Play
  • 1Password or Microsoft Authenticator — both work if you already have them

Recommended: Authy

Authy backs up your codes to the cloud, so if you switch phones you don't lose access. Google Authenticator doesn't do this by default.

B. Create a GitHub account

  1. Open your browser and go to https://github.com/signup
  2. Use your TELUS Digital work email — the one ending in @telusinternational.com.
  3. Pick a username and a strong password. Verify your email when GitHub sends you a code.
  4. Enable two-factor authentication when prompted:
    • Choose "Authenticator app" when given the option.
    • Open Authy (or your chosen app) on your phone and scan the QR code GitHub shows.
    • Type the 6-digit code from the app back into GitHub to confirm.
    • Save the recovery codes GitHub gives you — store them somewhere safe (a password manager is ideal). These are your emergency way back in if you ever lose your phone.
  5. Wait for the invitation email from GitHub — it will be titled "You've been invited to join the willowtreeapps organization". Click the "Accept invitation" button inside that email before continuing.

If the email never arrives

Check your spam folder first. If you still don't see it after a few minutes, the repository owner may need to re-send the invite from the GitHub org settings.

C. Download and set up Claude Desktop

You'll need the Claude Desktop application for the Salesforce MCP to work. If you don't have it yet, follow these steps:

  1. Open your browser and go to https://claude.ai/download
  2. Click the download link for Mac (it will download a .dmg file).
  3. Open the downloaded file and drag the Claude app into your Applications folder.
  4. Open Applications (press ⌘ + Space, type "Applications", press Enter) and double-click Claude to launch it.
  5. When prompted to log in, use your TELUS Digital work email — the one ending in @telusinternational.com.
  6. Complete the login process in your browser.

Leave Claude open — you'll use it after the setup is complete.

D. Open Terminal

You'll run several commands in a program called Terminal that comes pre-installed on every Mac.

  1. Press ⌘ + Space to open Spotlight.
  2. Type Terminal and press Enter.
  3. A black or white window opens with a blinking cursor. That's where you'll paste the commands below.

Copy & paste tip

Use the Copy button on each code block below, then in Terminal press ⌘ + V to paste, and Enter to run.

1

Install Homebrew

Homebrew is a free tool that lets you install other software with one line. We need it to install Node, Git, and a few other things.

Paste this into Terminal and press Enter:

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

You will be asked for your Mac login password — type it (you won't see any characters appear; that's normal) and press Enter. The installer takes 5–10 minutes.

When it finishes, it will print a section called "Next steps" with two commands you must run. They will look something like the lines below.

Copy what your screen shows

The exact path may vary on different Macs. If Terminal printed different commands under "Next steps", use those, not these.

echo 'eval "$(/opt/homebrew/bin/brew shellenv)"' >> /Users/$(whoami)/.zprofile
eval "$(/opt/homebrew/bin/brew shellenv)"

Run both of those in order, then verify Homebrew is working:

brew --version

You should see a version number (e.g. Homebrew 4.x.x). If you do, you're good.

2

Install Git, Node, GitHub CLI, and Salesforce CLI

One command installs Git, Node, and the GitHub CLI:

brew install git node gh

This takes 3–5 minutes. Wait for it to finish completely.

Then install the Salesforce CLI:

npm install -g @salesforce/cli

Verify everything is installed — each command below should print a version number:

git --version
node --version
gh --version
sf --version

If you see "command not found"

Close Terminal completely (⌘ + Q), reopen it, and try the command again. New tools sometimes only appear in fresh Terminal windows.

3

Sign in to GitHub through your browser

This connects your Terminal to your GitHub account so you can download the project. The login happens in your browser — no passwords typed into Terminal.

gh auth login --web --hostname github.com --git-protocol https

You'll be asked one question:

Authenticate Git with your GitHub credentials? (Y/n)

Type Y and press Enter.

Next, Terminal will display something like:

! First copy your one-time code: XXXX-XXXX
Press Enter to open github.com in your browser...
  1. Copy the one-time code shown on your screen (highlight it and press ⌘ + C).
  2. Press Enter in Terminal.
  3. Your browser opens to a GitHub page asking for the code. Paste it and click Continue.
  4. GitHub will ask you to confirm with your authenticator app (Authy / Google Authenticator). Approve the prompt.
  5. Click Authorize github.
  6. Return to Terminal — it will say ✓ Authentication complete.

Now connect Git itself to your GitHub login. This tells Git to use gh whenever it needs to authenticate to GitHub — so git clone and git pull "just work" without ever asking for a password.

gh auth setup-git

It will print nothing on success — that's correct.

Verify everything is connected:

gh auth status

You should see your GitHub username listed as logged in, plus a line that says "Git operations protocol: https".

4

Download the project

The guide saves the project to your Desktop for simplicity. You can pick any folder, but use the same path in every step below.

Choose one of the two paths below:

Path A — Standard (WillowTree organization)

Use this if you have access to the WillowTree GitHub organization.

cd ~/Desktop
git clone https://github.com/willowtreeapps/salesforce-mcp.git

Path B — Alternative (Mohammed's repository)

Use this if you cannot access the WillowTree organization. This is an alternative maintained by Mohammed (GitHub username: makwt).

cd ~/Desktop
git clone https://github.com/makwt/salesforce-mcp.git

Need collaborator access?

If you use Path B and don't have access, provide Mohammed (makwt) with your GitHub username and he can grant you access to the repository.

After cloning (both paths)

If this is your first time using git in Terminal, it may ask for your name and email. Run these two commands first — replace the placeholders with your real name and TELUS Digital email:

git config --global user.name "Your Full Name"
git config --global user.email "your.name@telusinternational.com"

Then move into the folder you just cloned (the folder name is the same for both paths):

cd ~/Desktop/salesforce-mcp
5

Run the setup script

This is the one command that does all the heavy lifting — it installs dependencies, builds the project, configures Claude, and optionally logs you into Salesforce.

bash setup.sh

The script will:

  1. Check prerequisites — confirms Node and Salesforce CLI are installed (they are, from Step 2).
  2. Install and build the project — this takes 1–2 minutes; you'll see progress in Terminal.
  3. Ask which AI client you use — a small dialog box pops up. Pick:
    • Claude Desktop / Cowork — if you use the Claude desktop app
    • Claude Code — if you also use the Claude Code terminal tool
    You can pick more than one by holding ⌘ while clicking each one. Click OK.
  4. Offer to log in to Salesforce — another dialog box. Click Yes.

When you click Yes on the Salesforce login:

  • Your browser opens to the WillowTree Salesforce login page.
  • Log in with your normal WillowTree SSO credentials (the same ones you use for Salesforce in the browser every day).
  • Approve any access prompts.
  • Return to Terminal — it will say "Authentication complete — your session is ready."

You're done

You'll see a green "Setup complete!" banner in Terminal. One last step.

6

Restart Claude and test

Close and reopen whichever client you configured:

  • Claude Desktop / Cowork — quit (⌘ + Q) and reopen.
  • Claude Code — start a brand new session in Terminal.

Then ask Claude a test question, like:

"What projects am I currently on?"

or

"Show me the revenue forecast for this quarter."

If Claude returns Salesforce data, you're done.

Common questions

My session expired / I get an authentication error

Re-authenticate at the terminal with one command, then tell your AI assistant "reconnect":

sf org login web --instance-url https://willowtree.my.salesforce.com --alias willowtree

Your browser opens to the WillowTree Salesforce login page (same as during setup). Log in, return to Terminal, then tell your AI assistant: "reconnect". It will pick up the new session without restarting anything.

I want to re-run setup

That's safe. Navigate back to the folder and run setup again:

cd ~/Desktop/salesforce-mcp
bash setup.sh

I want to update the project later when there are changes

cd ~/Desktop/salesforce-mcp
git pull
bash setup.sh

I lost my phone / I can't get past the GitHub MFA prompt

Use one of the recovery codes you saved when you set up MFA. Go to github.com/login, enter your password, and on the MFA prompt click "Use a recovery code". Once logged in, set up MFA again on your new phone.

Something broke / I'm stuck

Copy and paste whatever Terminal printed (the full error message) and share it with your team or the repository maintainer. The error message is the single most useful thing for diagnosing problems.

Troubleshooting

A tool call hangs / never returns a result

The MCP server runs over a data pipe and cannot open a browser. If a tool call seems stuck and never returns, you are likely on an older version of this repo that tried to launch a browser from inside the server. Pull the latest and rebuild:

cd ~/Desktop/salesforce-mcp
git pull
bash setup.sh

Then restart your AI client (quit and reopen). The server now reads your existing Salesforce CLI session silently — no browser needed after the initial setup.

Claude Code picks the wrong Salesforce tool (not returning my data)

Claude Code has access to several Salesforce-related connectors. If it reaches for one called "TDX Salesforce" or "WT Salesforce Sandbox" instead of your local server, those are separate cloud connectors that require their own OAuth setup and are unrelated to this MCP.

Fix: be explicit in your prompt:

"Use the local salesforce MCP — what projects am I on?"

Or name the tool directly, e.g. "call list_my_projects". The local tools are named things like list_my_projects, whoami, get_revenue_forecast, and find_contacts.
